# T7.1 · Setting Up a Sensor Probe with ESP32

**Location:** `textbook/7/realisation.tex:123–126`
**Textbook context:** Hardware layer; sensor selection; sampling rate; power budget.
**Data Foundry features:** IoT Dataset, ESP32 MicroPython client
**Platform pages:** [ESP32 guide](https://data-foundry.net/Guides/Examples/ESP-32/index.html) · [MicroPython](https://data-foundry.net/Guides/Examples/ESP-32/ESP32micropython.html) · [ESP tools](https://data-foundry.net/Guides/Tools/ESPtools/MicropythonSuite.html)

## Steps

1. **Hardware prep** – Flash MicroPython onto an ESP32. Connect a DHT22 (temperature/humidity) or PIR motion sensor.
2. **Data Foundry IoT Dataset** – Create an IoT Dataset in your project; note the dataset key.
3. **Firmware** – Using the Data Foundry MicroPython client, write a loop that reads the sensor and POSTs a data point every 10 seconds.
4. **Verify** – Check the IoT Dataset dashboard in real time. Trigger the sensor manually and confirm data arrives.
5. **Power budget** – Activate deep sleep between readings; measure current draw with and without sleep. Estimate battery life for a 7-day deployment.
6. **Deployment checklist** – Test power-interruption recovery, timestamp synchronisation (NTP), and sensor range validity before fielding.
