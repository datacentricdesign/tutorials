# T7.4 · Building a Simulated Data Interface

**Location:** `textbook/7/realisation.tex:229–232`
**Textbook context:** Simulated prototypes; pre-recorded data; Streamlit or p5.js; walking participants through a simulation.
**Data Foundry features:** Existing Dataset (hosts CSV), Website Hosting, JavaScript client
**Platform pages:** [My First Data Foundry Website](https://data-foundry.net/Tutorials/GettingStarted/MyFirstDataFoundryWebsite.html) · [JavaScript client](https://data-foundry.net/Guides/Clients/Javascript.html) · [Existing Dataset](https://data-foundry.net/Learning/Datasets/DatasetsOverview/existingdataset.html)

## Steps

1. **Upload dataset** – Export a CSV from an earlier study (e.g. the IoT data from T3.3). Upload to an Existing Dataset in Data Foundry.
2. **Choose interface tool** – Streamlit (if your pipeline is in Python) or p5.js / vanilla HTML+JS (if deploying as a web page). For Data Foundry hosting, use HTML+JS with the JavaScript client.
3. **Build the shell** – Load the CSV and render one chart (line chart, bar chart, or summary panel). Use an LLM to generate the initial HTML/JS from a concrete description of the interface (see T7.6).
4. **Host on Data Foundry** – Upload the HTML file to Data Foundry website hosting. Share the URL with a colleague to test remote access.
5. **Run a walkthrough session** – Present the simulation to a participant as though it were a live product. Observe their reactions; note questions and misunderstandings.
6. **Reflect** – What value of the product concept did the simulation test? What remained untestable without real-time data?
