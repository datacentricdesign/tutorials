# T7.4 · End-to-End Stack: Sensor → Data Foundry → Interface

**Location:** `textbook/7/realisation.tex:248–251`
**Textbook context:** Live data products; remote iteration; data quality in the wild.
**Data Foundry features:** IoT Dataset, JavaScript/Python client, Starboard Notebook, Website Hosting
**Platform pages:** [From Prototype to Website](https://data-foundry.net/Tutorials/GettingStarted/FromPrototypeToWebsite.html) · [IoT Dataset](https://data-foundry.net/Learning/Datasets/DatasetsOverview/iotdataset.html)

## Steps

1. **Sensor layer** – Use the probe from T7.1 to stream live data into a Data Foundry IoT Dataset.
2. **Processing layer** – Add a Data Foundry Script (or Starboard Notebook) that reads new data points, applies a simple classification (e.g. "high activity" / "low activity"), and writes results back to an Entity Dataset.
3. **Interface layer** – Build a participant-facing HTML page using the JavaScript client that reads the Entity Dataset and displays the current status as a simple ambient indicator (text label, colour, icon).
4. **Host & deploy** – Host the interface on Data Foundry. Share the link with a participant for a 24-hour live test.
5. **Monitor & iterate** – Check the IoT Dataset for missing data, anomalies, and timestamp issues. Make one interface change remotely without visiting the participant.
6. **Reflect** – What failed in the wild that worked in the lab? How did participants engage differently with real personal data versus a simulation?
