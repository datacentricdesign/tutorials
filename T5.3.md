# T5.3 · Text Data Analysis

**Location:** `textbook/5/analytics.tex:342–345`
**Textbook context:** RTA phases 4–6; sentiment analysis; topic modelling; hybrid workflow.
**Data Foundry features:** Form/Diary/Existing Dataset, Starboard Notebook, Python client
**Platform pages:** [Annotation Dataset](https://data-foundry.net/Learning/Datasets/DatasetsOverview/annotationdataset.html) · [Starboard Notebook](https://data-foundry.net/Guides/Tools/Starboard/StarboardNotebook.html)

## Steps

1. **Review candidate themes (RTA Phase 4)** – Load the annotated corpus from T4.3. Test each candidate theme against the full dataset: does it hold? Split, merge, or dissolve themes as needed.
2. **Define themes (RTA Phase 5)** – Write a definition sentence and name for each surviving theme. Select 2–3 representative quotes.
3. **Sentiment analysis** – Apply a lexicon-based sentiment scorer (e.g. VADER) to each text item in a Starboard Notebook. Plot sentiment distribution and trend over time. Read the 10 most negative and 10 most positive items manually. Does the score match your reading?
4. **Topic modelling** – Run NMF or LDA on the preprocessed corpus (lowercased, stopwords removed). Try topic counts of 5, 10, and 20. Compare the top words per topic against your RTA themes.
5. **Hybrid synthesis** – Identify where computational findings confirm, extend, or challenge the qualitative themes. Write a 1-page synthesis (to become the written analysis, RTA Phase 6).
6. **Reflect** – Which method was more generative for design insight: RTA or topic modelling? What would have been lost if you had only used one?
