## Step 1 — Summary statistics with design intent

### Compute daily activity per participant

Aggregate from minute-level to daily means. This gives one number per participant per day — the natural unit for comparing activity across people.

```python
import pandas as pd
import numpy as np

daily = (
    df.assign(date=df["ts"].dt.date)
    .groupby(["participant", "date"])["accel_std"]
    .mean()
    .reset_index()
)
daily.columns = ["participant", "date", "daily_mean"]
```

### Characterise each participant

Three statistics together describe the design-relevant shape of a participant's activity:

- **Median daily activity** — the robust centre, not pulled by occasional high-activity days
- **Coefficient of variation** (std / mean) — how consistent the participant's day-to-day activity is. Low CV means regular habits; high CV means variable or unpredictable behaviour.
- **10th–90th percentile range** — the span of "normal" for this person, excluding extremes

```python
def cv(x):
    return x.std() / x.mean() if x.mean() > 0 else np.nan

summary = (
    daily.groupby("participant")["daily_mean"]
    .agg(
        median = "median",
        cv     = cv,
        p10    = lambda x: x.quantile(0.10),
        p90    = lambda x: x.quantile(0.90),
    )
    .assign(p10_p90_range=lambda d: d["p90"] - d["p10"])
    .round(4)
)
print(summary)
```

Look at the output and note:

- Which participant has the **highest median activity**? Which is most sedentary on average?
- Which has the **highest CV**? This participant's behaviour is least predictable day-to-day — their product experience needs to handle variability rather than assuming a stable routine.
- Which has the **narrowest 10–90 range**? This participant is the most consistent — a product with a fixed daily prompt might serve them well.

This is statistical description as design thinking: each number points toward a different product decision.

## Step 2 — Weekday vs. weekend

One of the most informative comparisons for behaviour data is weekday versus weekend. If a participant's activity shifts significantly between the two, any connected product that assumes a uniform weekly routine will fail them on at least one type of day.

```python
import matplotlib.pyplot as plt

df["day_type"] = df["ts"].dt.dayofweek.apply(
    lambda d: "Weekend" if d >= 5 else "Weekday"
)

fig, axes = plt.subplots(1, len(participants), figsize=(14, 4), sharey=True)

for ax, pid in zip(axes, participants):
    part    = df[df["participant"] == pid]
    weekday = part[part["day_type"] == "Weekday"]["accel_std"].dropna()
    weekend = part[part["day_type"] == "Weekend"]["accel_std"].dropna()
    bp = ax.boxplot(
        [weekday, weekend],
        labels=["Wkday", "Wkend"],
        patch_artist=True,
        medianprops=dict(color="white", linewidth=2),
        flierprops=dict(marker=".", markersize=2, alpha=0.3),
    )
    bp["boxes"][0].set_facecolor("steelblue")
    bp["boxes"][1].set_facecolor("darkorange")
    ax.set_title(pid, fontsize=10)

axes[0].set_ylabel("accel_std (g)")
fig.suptitle("Activity level — weekday vs. weekend per participant", fontsize=12)
plt.tight_layout()
plt.show()
```

For each participant, ask:
- Is the **median** higher on weekdays or weekends?
- Is the **spread** (box height) different between the two? A wider box on weekends suggests less structured time.
- Are there participants where the distributions are nearly identical — suggesting their activity is independent of the weekday/weekend structure?

{: .note }
> With only 7 days of data, some participants have as few as one or two weekend days. Sample sizes are unequal by design. Treat this as an exploratory comparison, not a statistical test.

## Step 3 — Daily activity trend

Does activity increase, decrease, or stay flat over the seven days? A directional change might signal adaptation to the study, a weekly energy cycle, or an event during the study period. Even with only 7 data points per participant, the analytical question is whether any trend is *large enough to matter for design* relative to typical day-to-day variation.

```python
from scipy import stats

# Assign a day index (0 = first day) within each participant's study
daily["day_num"] = daily.groupby("participant").cumcount()

_palette = ["steelblue", "darkorange", "seagreen", "crimson", "mediumpurple"]
colors = dict(zip(participants, _palette))

fig, ax = plt.subplots(figsize=(12, 4))

for pid in participants:
    p = daily[daily["participant"] == pid]
    ax.plot(p["day_num"], p["daily_mean"], "o-",
            label=pid, color=colors[pid], lw=1.5, ms=5)
    slope, intercept, *_ = stats.linregress(p["day_num"], p["daily_mean"])
    x = np.array([p["day_num"].min(), p["day_num"].max()])
    ax.plot(x, intercept + slope * x, "--", color=colors[pid], lw=0.8, alpha=0.6)

ax.set_xlabel("Day of study (0 = first day)")
ax.set_ylabel("Daily mean accel_std (g)")
ax.set_title("Daily activity trend over 7 days — all participants")
ax.legend(fontsize=9)
plt.tight_layout()
plt.show()
```

Print the slopes and compare each to the participant's own day-to-day variability:

```python
header = f"  {'ID':>4}  {'slope (g/day)':>14}  {'7-day change':>13}  " \
         f"{'daily std':>10}  {'ratio':>7}  {'notable?':>9}"
print(header)
print("  " + "-" * 62)

for pid in participants:
    p = daily[daily["participant"] == pid]
    slope, intercept, *_ = stats.linregress(p["day_num"], p["daily_mean"])
    total   = slope * 6           # change over days 0–6
    std_d   = p["daily_mean"].std()
    ratio   = abs(total) / std_d if std_d > 0 else 0
    notable = "yes" if ratio > 0.5 else "no"
    print(f"  {pid:>4}  {slope:>+14.5f}  {total:>+13.5f}  "
          f"{std_d:>10.5f}  {ratio:>7.2f}  {notable:>9}")
```

A trend is worth investigating when its total magnitude over the study period exceeds roughly half the typical day-to-day variation (ratio > 0.5). For participants where the trend is notable, the analytical question is: *what might explain it?* The textbook calls this the *interpretable* criterion — a meaningful trend must be linkable to a plausible cause or context.

## Step 4 — Correlation: movement and location

### Fix the watch unit mismatch

In T4.1 Step 1 you noticed that `watch_accel_mean` values are approximately 1 000× larger than `accel_mean`. Both measure accelerometer magnitude, but one is in g-units and the other in milli-g. Before computing correlations, bring the watch signal into the same units as the phone:

```python
df["watch_accel_g"] = df["watch_accel_mean"] / 1000
```

### Scatter plots across participants

Two correlations have direct design implications:

- **Phone vs. watch signal** — do both sensors agree about movement intensity? Strong correlation would suggest one is redundant; divergence would mean they capture different aspects of behaviour.
- **Movement vs. GPS activity** — does `accel_std` predict `gps_updates`? A strong correlation would imply that "physically active" nearly always means "outside with GPS signal"; a weak one would mean the two are largely independent.

```python
fig, axes = plt.subplots(2, len(participants), figsize=(16, 6),
                         sharex="row", sharey="row")

for col, pid in enumerate(participants):
    part = df[df["participant"] == pid].dropna(
        subset=["accel_std", "watch_accel_g", "gps_updates"]
    )

    # Row 0: phone vs. watch
    ax = axes[0, col]
    ax.scatter(part["accel_std"], part["watch_accel_g"],
               alpha=0.1, s=3, color="steelblue")
    r_pw = part[["accel_std", "watch_accel_g"]].corr().iloc[0, 1]
    ax.set_title(f"{pid}\nr = {r_pw:.2f}", fontsize=9)
    if col == 0:
        ax.set_ylabel("watch_accel_g (g)", fontsize=9)
    ax.set_xlabel("accel_std (g)", fontsize=8)

    # Row 1: movement vs. GPS
    ax = axes[1, col]
    ax.scatter(part["gps_updates"], part["accel_std"],
               alpha=0.1, s=3, color="darkorange")
    r_pg = part[["accel_std", "gps_updates"]].corr().iloc[0, 1]
    ax.set_title(f"r = {r_pg:.2f}", fontsize=9)
    ax.set_xlabel("GPS updates", fontsize=8)
    if col == 0:
        ax.set_ylabel("accel_std (g)", fontsize=9)

axes[0, 2].set_title("Phone vs. watch signal\n" + axes[0, 2].get_title(),
                      fontsize=9)
fig.text(0.5, 1.01, "Phone vs. watch signal", ha="center", fontsize=11)
fig.text(0.5, 0.48, "Movement vs. GPS activity", ha="center", fontsize=11)
plt.tight_layout()
plt.show()
```

For each scatter plot, the correlation coefficient is only the starting point. Look at the shape:
- Is the relationship **linear**, or do the points form clusters that suggest different behavioural modes?
- Are there participants where the pattern looks qualitatively different from others?

The GPS correlation carries a direct design implication: if phone movement and GPS activity are weakly correlated, a product that infers outdoor exercise from accelerometer data alone would misclassify a significant portion of activity. That is not a data quality issue — it is a finding about how physical activity and location relate for these participants.

## Step 6 — Prepare PDA materials

You have computed a set of analytical findings for each participant. The next step is to translate this into visual materials that could be used in a Participatory Data Analysis session — shown to participants to invite their interpretation of patterns they may not have been aware of.

**Reminder:** The ExtraSensory participants are not reachable. What you produce here is a rehearsal of the format. The goal is to learn it well enough to apply it in your group assignment with participants you have access to.

### Individual data portrait per participant

A data portrait is a single A3 sheet that communicates one participant's analytical profile in plain language. It combines the weekly activity heatmap with a weekday/weekend comparison and a text panel of plain-language statistics — making three layers of analysis readable at a glance.

```python
import matplotlib as mpl
import matplotlib.gridspec as gridspec

mpl.rcParams["svg.fonttype"] = "none"
mpl.rcParams["font.family"]  = "DejaVu Sans"

A3_W = 420 / 25.4   # inches
A3_H = 297 / 25.4
DAY_ORDER = ["Monday", "Tuesday", "Wednesday",
             "Thursday", "Friday", "Saturday", "Sunday"]


def make_portrait(df, participant, output_path):
    """Generate an A3 landscape data portrait SVG for one participant."""
    part_df = df[df["participant"] == participant].copy()
    part_ts = part_df.set_index("ts").sort_index()

    # ── Heatmap pivot ──────────────────────────────────────────────────────
    hourly = part_ts["accel_std"].resample("1h").mean().to_frame()
    hourly["hour"]    = hourly.index.hour
    hourly["weekday"] = hourly.index.day_name()
    pivot = (
        hourly.groupby(["weekday", "hour"])["accel_std"]
        .mean().unstack("hour")
        .reindex([d for d in DAY_ORDER if d in hourly["weekday"].unique()])
    )

    # ── Weekday / weekend medians ──────────────────────────────────────────
    part_df["day_type"] = part_df["ts"].dt.dayofweek.apply(
        lambda d: "Weekend" if d >= 5 else "Weekday"
    )
    wd_med = part_df[part_df["day_type"] == "Weekday"]["accel_std"].median()
    we_med = part_df[part_df["day_type"] == "Weekend"]["accel_std"].median()

    # ── Plain-language stats ───────────────────────────────────────────────
    daily_p = (
        part_df.assign(date=part_df["ts"].dt.date)
        .groupby("date")["accel_std"].mean()
    )
    most_active = daily_p.idxmax().strftime("%A")
    quietest    = daily_p.idxmin().strftime("%A")
    cv_val      = daily_p.std() / daily_p.mean()
    regularity  = "high" if cv_val < 0.15 else ("medium" if cv_val < 0.30 else "low")
    gps_pct     = int(100 * part_df["gps_updates"].notna().mean())

    # ── Layout ────────────────────────────────────────────────────────────
    fig = plt.figure(figsize=(A3_W, A3_H))
    gs  = gridspec.GridSpec(
        2, 2, figure=fig,
        width_ratios=[2.2, 1], height_ratios=[1.6, 1],
        hspace=0.38, wspace=0.30,
        left=0.06, right=0.97, top=0.92, bottom=0.06,
    )
    ax_heat = fig.add_subplot(gs[:, 0])   # full left column
    ax_wdwe = fig.add_subplot(gs[0, 1])   # top right
    ax_stat = fig.add_subplot(gs[1, 1])   # bottom right

    # Heatmap
    im = ax_heat.imshow(pivot.values, aspect="auto", cmap="YlOrRd",
                        origin="upper", vmin=0, vmax=0.08)
    ax_heat.set_xticks(range(0, 24, 2))
    ax_heat.set_xticklabels([f"{h:02d}:00" for h in range(0, 24, 2)],
                             fontsize=8, rotation=45, ha="right")
    ax_heat.set_yticks(range(len(pivot)))
    ax_heat.set_yticklabels(pivot.index, fontsize=10)
    ax_heat.set_xlabel("Time of day", fontsize=11)
    ax_heat.set_title("Movement intensity — last week", fontsize=14, pad=10)
    plt.colorbar(im, ax=ax_heat, label="Movement intensity",
                 fraction=0.02, pad=0.02)

    # Weekday / weekend bar
    bar_h = max(wd_med, we_med)
    ax_wdwe.bar(["Weekdays", "Weekends"], [wd_med, we_med],
                color=["steelblue", "darkorange"], alpha=0.85)
    ax_wdwe.set_ylabel("Typical activity level", fontsize=9)
    ax_wdwe.set_title("Weekday vs. weekend", fontsize=11)
    ax_wdwe.set_ylim(0, bar_h * 1.5)
    ax_wdwe.tick_params(labelsize=9)

    # Plain-language statistics
    ax_stat.axis("off")
    stats_lines = (
        f"Most active day:    {most_active}\n"
        f"Quietest day:       {quietest}\n"
        f"Day-to-day routine: {regularity} consistency\n"
        f"GPS available:      {gps_pct}% of windows"
    )
    ax_stat.text(
        0.05, 0.88, stats_lines,
        transform=ax_stat.transAxes, fontsize=10,
        verticalalignment="top", fontfamily="monospace",
        bbox=dict(boxstyle="round,pad=0.5",
                  facecolor="#f5f5f5", edgecolor="#cccccc"),
    )
    ax_stat.set_title("At a glance", fontsize=11, loc="left", pad=8)

    fig.suptitle(f"Activity portrait — {participant}", fontsize=16, y=0.97)
    plt.savefig(output_path, format="svg", bbox_inches="tight")
    plt.close(fig)
    print(f"Saved {output_path}")


for pid in participants:
    make_portrait(df, pid, f"portrait_{pid}.svg")
```

## Step 7 — Workshop exercise: collective sense-making

This exercise rehearses the collective PDA format. It is designed for groups of four or five students, each representing one ExtraSensory participant. The structure is adapted from two research studies: Panagiotidou et al. (2023), who used printed data physicalisations and a shared reference frame to support collective negotiation around energy data, and Yan et al. (2025), who showed that alternating between individual and shared working spaces in data sensemaking surfaces insights invisible in solo analysis.

Each student acts as a **proxy** for their assigned participant — speaking on the basis of the data alone, without lived context. This constraint is the point: it makes visible exactly what the data can and cannot explain, and what questions only the participant could answer.

### Before the session

Each student brings:
- Their printed A3 portrait for one participant
- Two sticky notes prepared from Steps 1–5, each noting one thing they noticed that they cannot explain from the data alone

The group shares one printed A3 group comparison sheet, placed at the centre of the table.

### Round 1 — Solo annotation (10 minutes)

Working individually, annotate your own portrait with sticky notes before discussing anything with others:

| Colour | Prompt |
|---|---|
| Yellow | **"I notice…"** — a pattern you can describe from the data |
| Orange | **"I wonder…"** — something the data suggests but cannot confirm |
| Pink | **"This data cannot explain…"** — a gap that only the participant could fill |

Place at least three sticky notes. Keep your portrait face-up but do not discuss yet.

### Round 2 — Group comparison (15 minutes)

Place all individual portraits around the group comparison sheet. Work through these questions together:

1. **What do all five participants share?** Look for patterns present in every portrait — a quiet period at the same time of day, a similar broad weekday structure. These are candidate universal patterns: features a product could assume without segmenting users.

2. **Where does one participant stand out?** Find one cell, bar, or statistic that is clearly different for one person. Formulate a one-sentence hypothesis about what might explain it. This exercise practises moving from *describing* a difference to *interpreting* it.

3. **What becomes visible in comparison that was invisible in individual analysis?** This is the core question of the exercise. Patterns that look unremarkable in a single portrait often become significant when you can see that others look different. Note at least one example as a group.

The last question is the key finding of Yan et al. (2025): comparison with a partner regularly uncovered patterns that neither participant had noticed alone — precisely because the contrast made the pattern visible as a pattern rather than just a feature.

### Round 3 — Question drafting (10 minutes)

Individually, write three interview questions you would ask your participant if you had access to them — questions informed by what emerged in Round 2. Good PDA questions:
- Reference a specific pattern from the portrait: *"Your Saturday mornings are consistently more active than your weekday mornings — does that match how you experience your weekends?"*
- Follow up on a comparison-dependent observation: *"You are the only participant whose GPS activity does not increase on your most active days — can you help me understand what physical activity looks like for you?"*
- Surface a gap the data cannot fill: *"There are several periods of multiple hours where we have no data at all — do you know what was happening during those gaps?"*

Write these three questions in a text cell in your notebook.

### After the session

Collect all sticky notes from the individual portraits onto the group comparison sheet. Photograph the result. The annotations mark findings that emerged from collective comparison — keep this image as documentation of your analytical process.

{: .note }
> **Applying this to your group project.** In your own project, the roles are filled by your actual participants rather than proxies. Round 1 becomes your individual analysis before the session. Round 2 is conducted with participants present — you show each person their own portrait and ask them to interpret what they see, then introduce the group comparison sheet for a second phase where participants discuss similarities and differences. Round 3 becomes the interview guide you use in follow-up conversations. The structure is the same; only the people change.

## Reflect

Before applying this approach in your design project, consider:

1. **Statistics as descriptions of people.** The coefficient of variation labels one participant as having low routine consistency. That label is computed from seven days of data. What would you need to be confident that this characterisation holds over a month? Over a year? How might the label shape how you design for this person — and how might it mislead you?

2. **The comparison changes the question.** In Round 2 of the workshop, observations that seemed unremarkable in individual portraits became significant through comparison. What does this tell you about the analytical value of studying multiple participants rather than one? What is the minimum group size for this comparative effect to be useful in your own project?

3. **Correlation as a design hypothesis.** You examined the correlation between phone movement and GPS activity. Suppose you found a strong positive correlation. What product decisions would that support — and what assumption would be silently embedded in those decisions? What would break for a user whose movement and GPS patterns are not correlated?

4. **The proxy problem.** In the workshop, you acted as a proxy for your participant — speaking from data alone, without their lived context. How long was your list of questions in Round 3 relative to the things the data answered directly? What does that ratio tell you about where computational analysis stops and participatory analysis must begin?

5. **From rehearsal to practice.** This tutorial used ExtraSensory data as a stand-in. In your group project, you will collect data from participants you have access to. Which steps of this tutorial would you apply unchanged? Which would you adapt — and how? Are there steps you would add that this dataset could not support?
