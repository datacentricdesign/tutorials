# T3.3 · Sensor-Based Data Collection with Data Foundry

**Location:** `textbook/3/collection.tex:164–167`
**Textbook context:** Sensor-based collection; sampling frequency, duration, reliability.
**Data Foundry features:** IoT Dataset, ESP32 / Raspberry Pi clients, Device management
**Platform pages:** [IoT Dataset](https://data-foundry.net/Learning/Datasets/DatasetsOverview/iotdataset.html) · [ESP32 guide](https://data-foundry.net/Guides/Examples/ESP-32/index.html) · [Raspberry Pi guide](https://data-foundry.net/Guides/Examples/RaspberryPi/index.html)

## Steps

1. **Setup** – Create a Data Foundry project; add an IoT Dataset; register a device (ESP32 or mobile phone).
2. **First datalogger** – Follow the [My First Datalogger](https://data-foundry.net/Tutorials/GettingStarted/MyFirstDataLogger.html) tutorial to log a simple value (e.g. light level from a phone or temperature from an ESP32 DHT sensor).
3. **Sampling rate decision** – Configure the logging interval. Experiment with 1 s vs. 10 s vs. 60 s; observe the difference in data volume and resolution.
4. **Multi-sensor probe** – Add a second sensor channel (e.g. motion + temperature). Observe the IoT Dataset updating in real time on the Data Foundry dashboard.
5. **Deployment checklist** – Before a real-world deployment: verify readings against a reference, check battery life, test behaviour after power interruption.
6. **Reflect** – Which phenomena did the sampling rate capture? Which did it miss? How would you adjust for a week-long in-home deployment?
