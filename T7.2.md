# T7.2 · Connecting Sensors via MQTT to a Local Pipeline

**Location:** `textbook/7/realisation.tex:177–180`
**Textbook context:** WiFi/MQTT; Mosquitto broker; Telegraf → InfluxDB → Grafana stack.
**Data Foundry features:** IoT Dataset (as remote data sink / backup), OOCSI protocol
**Platform pages:** [IoT Dataset](https://data-foundry.net/Learning/Datasets/DatasetsOverview/iotdataset.html) · [My First Connected Datalogger](https://data-foundry.net/Tutorials/GettingStarted/MyFirstConnectedDataLogger.html)

## Steps

1. **Local stack** – Install Mosquitto (MQTT broker) and Grafana on a Raspberry Pi. Configure Telegraf to subscribe to an MQTT topic and write to InfluxDB.
2. **ESP32 publisher** – Modify the T7.1 firmware to publish sensor readings to `probe/room/temp` via MQTT instead of (or alongside) Data Foundry.
3. **Grafana dashboard** – Create a live panel showing the last hour of temperature readings.
4. **Data Foundry as cloud backup** – In parallel, push the same readings to a Data Foundry IoT Dataset via HTTP, giving remote access and persistent storage.
5. **Compare approaches** – Local stack: private, low-latency, offline-capable. Data Foundry cloud: accessible anywhere, managed, collaborative. Discuss trade-offs in the context of the home sensor study.
6. **Reflect** – What data should stay local? What benefits from being accessible remotely? How does this choice reflect your ethics commitments to participants?
