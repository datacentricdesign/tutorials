# T7.3 · Firmware, Logging, and Deployment Checklist

**Location:** `textbook/7/realisation.tex:206–209`
**Textbook context:** Building a data probe; four layers: sensing, logging, power, housing.
**Data Foundry features:** IoT Dataset, asynchronous ESP32 client
**Platform pages:** [Asynchronous ESP32](https://data-foundry.net/Guides/Examples/ESP-32/Asynchronous.html) · [ESP tools](https://data-foundry.net/Guides/Tools/ESPtools/MicropythonSuite.html)

## Steps

1. **Sensing layer** – Select sensor(s) and verify bench readings against a reference (thermometer, accelerometer tester). Document expected range and units.
2. **Logging layer** – Implement dual logging: SD card (offline backup) + Data Foundry IoT Dataset (live). Test that SD card writes survive a power interruption.
3. **Power layer** – Implement deep sleep. Measure and record actual current draw at each phase (active, sleep, transmission). Compute expected battery life.
4. **Housing layer** – 3D-print or assemble an enclosure appropriate to the deployment context. Test environmental robustness (drop, humidity if relevant).
5. **Pre-deployment checklist** – Run through: sensor validation ✓, timestamp sync ✓, battery life test ✓, power-interruption recovery ✓, participant briefing & consent ✓, data retrieval plan ✓.
6. **Reflect** – Which layer caused the most problems during testing? How would you change the design before the next deployment?
