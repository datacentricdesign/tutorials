# T7.6 · Visual Pipeline Design with Data Foundry Scripting

**Location:** `textbook/7/realisation.tex:320–323`
**Textbook context:** Data pipeline design; edge vs. cloud processing; research pipeline to production pipeline.
**Data Foundry features:** Scripting, IoT Dataset, Entity Dataset
**Platform pages:** [Scripting](https://data-foundry.net/Guides/Scripting/index.html) · [Entity Dataset](https://data-foundry.net/Learning/Datasets/DatasetsOverview/entitydataset.html)

## Steps

1. **Design the pipeline on paper** – Map the flow: raw sensor data → cleaning step → classification step → output to entity/dashboard. Identify where each step lives (device, gateway, cloud).
2. **Implement in Data Foundry Scripts** – Write a script that reads from an IoT Dataset, applies a threshold rule (e.g. "motion detected if value > 500"), and writes a labelled event to an Entity Dataset.
3. **Handle missing data** – Modify the script to detect gaps in the IoT stream (no data for > 2× the expected interval) and write a "gap" event to the Entity Dataset.
4. **Test with noisy data** – Inject intentional noise or gaps into the IoT Dataset; verify the pipeline handles them gracefully without crashing.
5. **Compare to Node-RED** – If Node-RED is installed on a Raspberry Pi, replicate the same pipeline visually. Compare the effort and transparency of each approach.
6. **Reflect** – At which step in the pipeline does data go from raw measurement to design-relevant information? Who should be involved in designing that step—engineers, designers, participants?
